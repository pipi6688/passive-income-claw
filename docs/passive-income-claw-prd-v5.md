# Passive Income Claw — 业务需求文档

**版本：** v5.1
**日期：** 2026-03-15

---

## 1. 产品定位

Passive Income Claw 是一个运行在 OpenClaw 上的 Binance 被动收益 AI 助手。

**核心机制：被动推送 + 按需执行**

系统定时自动扫描 Binance 生态的收益机会，发现匹配用户偏好的机会后主动推送通知。用户看到推送后决定是否执行，也可以随时主动发起咨询或下达执行指令。

```
定时扫描 → 发现匹配机会 → 推送通知
                                ↓
                    用户决定是否执行
                                ↓
               「帮我买」→ Agent 执行 → 结果反馈
```

**产品边界：**
- 不在用户授权范围外执行任何操作
- 不承诺收益，不替代用户决策
- 不执行提币操作

---

## 2. 业务背景

目标用户是 Binance 上大量长期持币、低频操作的用户。他们的核心痛点不是不知道有收益产品，而是：

- 没时间持续比较和关注
- 发现了机会还要自己手动操作，摩擦成本高
- 平台展示通用，不针对自己的持仓和偏好

纯推荐工具解决了前两条，但用户还是得记得去问。被动推送解决了"记得去查"的问题，执行闭环解决了"手动操作"的问题。

---

## 3. 目标用户

**主要用户**

长期 HODLer：持仓时间长，不想频繁操作，希望资产不要闲置，最理想是"设置一次系统自动帮我做"。

中级理财型用户：对 Binance 产品有基本了解，希望 AI 不只给建议，还能直接帮自己执行。

**次要用户**

新手用户：不熟悉各类收益产品，需要通俗解释。

资产不匹配用户：对某个机会感兴趣但持仓不符合参与条件，希望了解是否有其他参与路径。

---

## 4. 核心业务场景

### 场景一：首次配置

用户首次使用时，通过自然语言完成两件事：

1. 描述偏好 → 系统生成用户画像并持久化
2. 配置执行授权 → 设定执行开关、金额上限、确认方式等

配置完成后系统立即输出当前适合的推荐作为首次参考，并告知后续将定期自动推送。

---

### 场景二：系统定时推送（核心场景）

系统每日自动扫描，发现匹配机会后主动推送精简摘要：

```
【收益机会提醒】
根据你的偏好（稳健型 / BNB 持仓），今天发现 2 个值得关注：

1. BNB 活期理财 — 年化 5.4%，风险低，可随时赎回 ✅ 新上线
2. USDT 定期 30 天 — 年化 8.2%，较上周上涨 1.1% ↑

如需执行，直接告诉我：「帮我买第 1 个」
```

推送到此结束，执行由用户主动发起。

---

### 场景三：条件触发推送

以下情况无需等待定时周期，立即触发推送：
- 出现新机会且与用户画像匹配
- 已关注机会的收益率出现明显变化（超过设定阈值）
- 用户主要持仓资产出现新的适配机会

---

### 场景四：用户主动询问

用户随时可以主动发起询问，系统基于已有画像直接响应，无需重新描述偏好：

- 「现在有哪些适合我的机会？」
- 「帮我详细分析一下刚才推送的定期产品」
- 「帮我买第 1 个，500 USDT」

---

### 场景五：资产不匹配，路径分析

当用户持仓资产与目标机会不匹配时，系统对比三条路径并给出建议：

| 路径 | 逻辑 | 主要风险 |
|------|------|---------|
| 直接换仓参与 | 卖出原有资产换入目标资产 | 原仓位机会成本 |
| 抵押借贷参与 | 以原有资产抵押借入目标资产 | 利息侵蚀收益 + 清算风险 |
| 暂不参与 | 保持原有持仓不操作 | 错过当前机会 |

用户确认后可直接执行推荐路径。

## 5. 功能需求

### P0 — 必须实现

#### 5.1 用户画像建立与持久化

- 支持自然语言输入描述偏好
- 转化为结构化标签并持久化存储
- 支持随时更新

| 维度 | 可能取值 |
|------|---------|
| 风险偏好 | 保守 / 稳健 / 收益优先 |
| 操作频率 | 低频 / 中频 |
| 流动性要求 | 高 / 中 / 低 |
| 持仓限制 | 不卖出原有持仓 / 可换仓 |
| 主要持仓资产 | BTC / ETH / BNB / USDT 等 |

---

#### 5.2 执行授权配置

用户首次使用时配置，系统严格在此范围内执行。

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| 执行开关 | 是否允许系统执行操作 | 关闭 |
| 确认模式 | confirm-first（执行前确认）/ auto（全自动） | confirm-first |
| 单次金额上限 | 每次操作最大金额（USDT 计价） | 用户设定 |
| 单日累计上限 | 当日累计执行上限 | 用户设定 |
| 允许操作类型 | 理财申购 / 含赎回 / 含借贷 | 仅理财申购 |
| 资产白名单 | 可操作的资产范围 | 用户设定 |

auto 模式：发现匹配机会 → 直接执行 → 推送结果，无确认，无延迟。全自动模式下执行结果通知强制推送，不可关闭。

---

#### 5.3 收益机会候选池

维护 Binance 生态被动收益机会候选池，每个机会包含：

| 字段 | 说明 |
|------|------|
| 机会名称 | 产品名称 |
| 机会类型 | 活期理财 / 定期锁仓 / 流动性挖矿等 |
| 收益特征 | 预期收益区间（浮动 / 固定） |
| 风险简述 | 主要风险点 |
| 流动性特征 | 是否可随时赎回，锁定周期 |
| 所需资产 | 参与所需资产类型 |
| 适合人群 | 推荐对象 |
| 可执行性 | 是否支持 API 自动申购 |
| 最近更新时间 | 用于判断是否触发变化推送 |

---

#### 5.4 定时扫描与推送

- 默认每日一次定时扫描
- 扫描结果与用户画像匹配，筛选适合机会
- 对比上次快照，有变化才推送，无变化不打扰
- 推送为精简摘要版（1～3 条），结尾引导用户主动执行
- **推送结束即完成，不自动触发执行**

---

#### 5.5 执行引擎

执行是独立入口，由以下两种情况触发：
1. 用户看到推送后主动发起（「帮我买」）
2. 用户预先配置了全自动模式

执行前必须通过授权校验（顺序执行，任一不通过立即终止）：
1. 执行开关是否开启
2. 本次金额 ≤ 单次上限
3. 今日已执行 + 本次 ≤ 单日累计上限
4. 操作类型在允许列表内
5. 资产在白名单内

支持的执行操作（按风险从低到高）：

| 操作 | 风险等级 |
|------|---------|
| 理财产品申购 | 低 |
| 理财产品赎回 | 低 |
| 抵押借贷 | 中 |

现货买卖不在 MVP 范围内。

---

#### 5.6 异常处理

| 异常类型 | 处理方式 |
|---------|---------|
| API 调用失败 | 通知用户，不自动重试，等待指令 |
| 申购额度已满 | 通知用户，推荐备选机会 |
| 余额不足 | 通知用户当前可用余额 |
| 超出授权范围 | 说明具体原因，终止执行 |

---

#### 5.7 执行记录

- 每次执行完成后推送结果通知
- 用户可随时查询历史执行记录
- 记录：操作类型、金额、时间、结果

---

#### 5.8 主动询问响应

用户主动发起时，系统给出完整输出：

- 用户画像摘要（基于已存储画像）
- Top 3～5 推荐（含推荐理由、风险等级、注意事项）
- 配置建议（极保守 / 稳健平衡 / 收益增强，1～3 套）
- 风险解释（为什么适合 / 不适合，风险在哪里）
- 参与路径分析（资产不匹配时）
- 一句话总结

---

### P1 — 强烈建议实现

- **借贷参与深度分析：** 分析利息是否侵蚀收益、抵押率与清算风险；保守型用户默认不推荐借贷路径
- **推送频率设置：** 用户可设定每日 / 每周 / 仅重要变化
- **通俗化解释：** 向新手用更简单的语言解释术语和推荐原因

---

### P2 — 后续迭代

- 收益追踪 Dashboard
- 再平衡建议
- 执行结果导出

## 6. 非功能需求

| 维度 | 要求 |
|------|------|
| 安全性 | 严格在授权范围内执行；超出授权立即拒绝 |
| 可理解性 | 推送版精简；主动问版完整；均避免堆砌术语 |
| 执行可靠性 | 异常有明确通知，不静默失败 |
| 合规表达 | 避免收益承诺和绝对化结论，附带风险提示 |
| 推送节制 | 无变化不推送，避免打扰 |

---

## 7. 成功标准

- 「设置一次，它就一直帮我盯着，不用我记得去查」
- 「推送的内容刚好是我关心的，不是什么都推」
- 「它帮我执行了，我知道它在什么范围内操作，我是放心的」

---

## 8. MVP 范围

**本次必须实现：**
- 用户画像建立与持久化
- 执行授权配置
- 收益机会候选池
- 定时扫描与推送
- 条件触发推送
- 执行引擎（理财申购 / 赎回）
- 异常处理与结果通知
- 执行记录
- 主动询问完整响应
- 资产不匹配路径分析与执行

**本次明确不做：**
- 现货买卖
- 收益追踪 Dashboard
- 再平衡建议
- 提币相关任何操作
